Put your Server in Production Mode

By default your server environment variables are not going to be in "production" mode. This is important because setting your server to be in production mode will allow you to:

  • Cache your VIEWS
  • Cache your CSS files
  • Generate less verbose errors
Production mode will allow your server to run faster.

Use a server Process Manager

When your app crashes you will need a Process Manager to restart your app and keep it online. Luckily using a Process Manager is very simple.
There are a number of Process Managers that you can use:

  • PM2 (recommended)
  • Forever
  • StrongLoop



Cluster your App instances

If your server has more than one cpu or core cpu then you can run an instance of your app on each CPU. This is a great way to scale your app for speed and reliabilty.

There are two ways you can custer your app:

  • Use a Process Manager (currently recommended)
  • Use the Cluster function in Nodejs
